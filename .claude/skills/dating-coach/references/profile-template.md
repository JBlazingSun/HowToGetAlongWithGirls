# 情况记录（每个女孩一份"活案卷"）

> 🌐 **语言**: 中文（当前） · [English](./profile-template.en.md)

教练做任何分析/建议前，先掌握目标女孩的情况记录。它不是一次性填完的静态表格，而是一份**随对话持续更新的案卷**：上半部是基本档案，下半部是**整体局势分析 + 时间线**。信息越全、越新，建议越精准。支持**同时管理多个对象**——每个女孩一份。

## 为什么要建

- 同样一句"在干嘛"，对 22 岁活泼的学妹和 30 岁高冷的职场女，回法完全不同。
- 阶段、雷点、她的兴趣点决定话术的力度和方向。
- 跨多次对话不断片：上次聊到哪、阶段怎么变的、踩过什么坑，一目了然。
- 同时认识几个女孩时，分开记，避免串味、各自推进。

## 记录模板

```markdown
# 情况记录：<代号或昵称>            （更新于：<日期>）

## 一、基本档案
- 代号/称呼：            （没真名用代号，如"奶茶店妹""舞蹈班A"）
- 年龄 / 职业身份：
- 怎么认识的：           （朋友介绍 / 同事同学 / 社交软件 / 搭讪 / …）
- 认识多久 & 线下见过几次：
- 性格：                 （高冷 / 活泼 / 慢热 / 强势 / 文静 / 作 / …）
- 爱好 / 喜欢的东西：     （越具体越好：某类宠物、某运动、某类食物…）
- 颜值 / 受追捧程度：     （影响"被追惯性"和废测频率）
- 你们的条件对比：       （大致谁更有优势，决定策略基调）
- 已知雷点 / 在意的事：   （前任、家庭、忌讳话题、价值观红线…）
- 你的目标：             （短期约会 / 认真谈 / 还在观察 …）

## 二、整体局势分析（核心，每次更新）
- **当前阶段**：          吸引 / 平淡 / 暧昧 / 恋爱 / 长期 …
- **她的兴趣**：          约 __ 成，趋势 ↑/→/↓
- **整体判断**：          一两句话说清现在是什么局面、为什么（吸引够不够、谁在推、卡在哪）
- **当前卡点**：          这一步最大的障碍是什么
- **下一步策略**：        接下来 1–3 步往哪带

## 三、时间线（倒序，新的在上）
- <日期>：发生了什么 / 她说了什么关键话 / 阶段或兴趣的变化 / 采取了什么动作、效果如何
- <日期>：……
```

## 如何使用

1. **首次聊到某个女孩**：用户没给信息就**先用一两句话问关键项**——至少要到「年龄、怎么认识的、目前阶段、她最近的反馈」，其余在对话中补全。**别一次抛一长串表格逼问**，缺什么问什么。
2. **每次作答**：用记录校准建议；回复开头按 SKILL.md 规则**注明当前阶段**（取自"二、整体局势分析"）。
3. **答完即更新**：把这次的新信息、新判断、阶段/兴趣变化，并入"二、整体局势分析"，并在"三、时间线"加一条。**记录始终反映最新局势**。
4. **多个对象**：分别记，用代号区分，绝不把 A 的情况套到 B 身上。

## 持久化：项目本地知识库 `profiles/`

动态情况记录有一个固定的家——项目根目录下的 `profiles/`：

```
profiles/
├── _index.md     # 全局总览表（所有对象 + 阶段/兴趣/趋势一览）
├── _template.md  # 新建记录时复制它
└── <代号>.md      # 每个女孩一份情况记录
```

工作方式：
- **新认识一个**：复制 `_template.md` 为 `<代号>.md`，填档案；在 `_index.md` 加一行。
- **每次提到某人**：先 `Read` 回 `profiles/<代号>.md` 再分析；答完把更新写回该文件，并**同步 `_index.md` 里这一行**（阶段/兴趣/趋势/最近更新）。
- **想看全局**：打开 `_index.md`，一眼看清手上每个人走到哪、该推进还是该止损。
- 首次为某人建档、或要写文件前，先征得用户同意；用户可用代号而非真名。

> 这是用户的私人数据，注意尊重隐私；除非用户要求，不主动外传或写入 `profiles/` 以外的位置。
